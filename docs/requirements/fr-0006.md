# FR-0006 — Show tokens-used% against time-elapsed% (pace) per window

For each window, the system shall show both the percentage of tokens consumed and
the percentage of the window's time elapsed, and indicate the **pace** between
them — whether tokens are being spent faster or slower than the clock.

- Time-elapsed% = (window duration − time-to-reset) ÷ window duration. Window
  durations: 5 hours and 7 days.
- Pace classifies `used% − elapsed%`: at or below the clock = on track; modestly
  ahead = watch; far ahead = burning hot (will run dry before reset).

This lets a high token percentage be read against how much of the window is gone:
20% tokens at 33% time elapsed is comfortable; 20% at 5% is not.
