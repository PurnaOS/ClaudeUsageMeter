# FR-0001 — Acquire per-window consumption and limit figures

The system shall obtain, for the current Claude Code subscription, the percentage
consumed and the reset time for each rate-limit window (5-hour and weekly),
refreshed on a regular interval.

## Mechanism (resolved by SPIKE-001, confirmed against a live 200)

- Call `GET https://api.anthropic.com/api/oauth/usage` with headers
  `Authorization: Bearer <token>` and `anthropic-beta: oauth-2025-04-20`.
- **Token source (corrected):** read `claudeAiOauth.accessToken` from the macOS
  **Keychain** item `Claude Code-credentials` (`security find-generic-password
  -s "Claude Code-credentials" -w`) — that is where Claude Code actually stores
  it. Fall back to `~/.claude/.credentials.json` (`CLAUDE_CONFIG_DIR`-aware) for
  non-macOS / file-based setups. Read-only; never refreshed or logged.
- **Response shape (corrected, verified live):** top-level `five_hour` and
  `seven_day` objects, each `utilization` (0–100 percent) and `resets_at` (an
  ISO-8601 string, not epoch). A richer `limits` array is also present but unused.
- The endpoint is rate-limited (HTTP 429). Cache the last good response to disk
  with a TTL; serve the cached value on any error (covered by [NFR-0002](nfr-0002.md)).
- Subscription (OAuth) accounts only. API-key-only users have no OAuth token and
  get an "unavailable" state.
