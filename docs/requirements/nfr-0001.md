# NFR-0001 — Negligible idle resource footprint

As an always-on app, the system shall consume negligible CPU and memory at idle
and shall poll its data source on a low-frequency interval, so running it
continuously has no noticeable cost to the machine.

## Verification — by design review (not an automated test)

Accepted on design grounds rather than a perf-test artifact: the app does no work
between polls (a single 60s `Timer`, no busy loop), each poll is one small HTTPS
request whose result is held in memory, and it runs as an `.accessory` agent with
no window rendering when closed. If idle cost ever becomes a concern, add a
measured test (sample CPU/RSS of the running agent over a few minutes against a
threshold) and link it here via `verified_by`.
