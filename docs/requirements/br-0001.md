# BR-0001 — Show % of both rate-limit windows consumed

The user must be able to see the percentage of tokens consumed against **both**
the Claude Code 5-hour limit and the weekly (7-day) limit, as live values that
reflect recent usage. (Subscription rate limits are 5-hour and weekly; there is
no 1-hour subscription window.)

The per-window consumption and limit figures come from the authenticated OAuth
usage endpoint — see [FR-0001](fr-0001.md), resolved by
[SPIKE-001](../work/spike-001.md).
